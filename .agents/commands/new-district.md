# New District

Districts should normally come from `supabase/seed.sql`, generated by `npm run db:seed:generate` from INEC UGED 2024. Do not hand-insert official Costa Rica geography unless the user is adding a non-official test/locality record.

When the user wants to add or verify a district:

1. Ask for province, canton, and district name, plus official 5-digit district code if known.
2. Check whether it already exists in `districts` joined through `cantons` and `provinces`.
3. Prefer regenerating the seed from `supabase/sources/inec-uged-2024.dbf`.
4. If manual SQL is explicitly needed, use the current schema with `provinces -> cantons -> districts`.
5. Confirm the expected URL path `/{canton-slug}/{district-slug}/`.

```sql
select
  provinces.name as province_name,
  cantons.name as canton_name,
  districts.name as district_name,
  districts.code,
  districts.slug
from districts
join cantons on cantons.id = districts.canton_id
join provinces on provinces.id = cantons.province_id
where cantons.slug = '{canton-slug}'
  and districts.slug = '{district-slug}';
```

```sql
insert into districts (canton_id, code, postal_code, name, slug, source)
select id, '{district-code}', '{district-code}', '{district}', '{district-slug}', 'manual'
from cantons
where slug = '{canton-slug}'
on conflict (postal_code) do nothing;
```
